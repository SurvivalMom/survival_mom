# きろくわけ — セットアップ手順

「きろくわけ」は、職員が気づいたことを書く（または音声入力する）だけで、
**介護記録 / 看護記録 / 申し送り / ヒヤリハット / その他** に自動でタグ付けし、
複数の職員が「申し送り送信」ボタンを押すと、全体の記録（一覧画面）に残る仕組みです。

既存の `kizuki.html`（気づきメモ）・`jikolog.html`（事故ログ）と同じ基盤
（Cloudflare Workers + KV、アクセスコード方式）を使っています。新しいサーバーや
Google/Stripe APIの本格連携は不要で、追加の運用コストは発生しません。

## 1. 自動分類の仕組み（コスト0円）

AI APIを呼ばず、キーワード辞書による分類を行っています（`kirokuwake.html` 内
`KEYWORD_DICT`）。1件の文章から複数カテゴリに同時にタグ付けされます。
職員は分類結果をタップして自由に追加・解除できるので、誤判定があっても
その場で修正できます。

辞書に語句を追加・修正したい場合は、`kirokuwake.html` の `KEYWORD_DICT` を
編集してください（コード変更のみで、サーバー作業は不要です）。

将来AIによる高精度分類を追加したい場合は、`apiPost` に `action:'ai_classify'`
のような新アクションを追加し、Worker側でAPIキーを使う形にすれば拡張できます。
ただしその場合はAPI利用コストが発生するため、現状は導入していません。

## 2. アクセスコード・購入の流れ（既存方式と同じ）

1. 既存の Cloudflare Worker（`cocolog-license.sr-g-solti.workers.dev`）に、
   `app: 'kirokuwake'` 用のアクセスコードを発行できることを確認してください。
   既存のWorkerが `app` をキーにしてコードを発行・検証する仕組みであれば、
   追加の実装作業は不要です。
2. 既存の `buy.stripe.com/...` Payment Linkと同じ要領で、購入用のリンクを
   1つ発行してください（金額・プラン名のみ変更）。
3. 購入後の案内文に、`kirokuwake.html` のURL + 発行したアクセスコードを
   記載して送付します（既存アプリの購入後フローと同一）。

## 3. デプロイ

`kirokuwake.html` は単一ファイルです。他のアプリと同様、現在ホスティングして
いる場所（GitHub Pages等）に置くだけで動作します。サーバーサイドの追加作業は
不要です。

## 4. 動作確認済み事項

- 文章入力 → 「分類する」→ キーワードに基づき複数カテゴリへ同時にタグ付け
- カテゴリの手動追加・解除
- 保存 → 一覧タブにカテゴリバッジ付きで表示、カテゴリ別フィルタ
- 「申し送り」タグの記録は「申し送り」タブに未送信一覧として表示
- 複数職員がそれぞれ「申し送り送信」を押すと、送信履歴（誰が・いつ・何件）が
  全体の一覧に残る
- 音声入力（Web Speech API、Chromeなど対応ブラウザのみ）

## 5. 今後の拡張候補（今回はスコープ外）

- AIによる高精度分類（要API契約・要コスト設計）
- Googleスプレッドシートへの自動転記（要Google連携実装）
- 施設・職員ごとの権限管理（要本格認証基盤）

これらが必要になった場合は、要件と費用負担方法を決めてから着手することを
お勧めします。
